# Deploying Volt Tackle on Render

Render hosts the **whole stack in one project** — frontend, backend, database,
and vector store — because (unlike serverless platforms) it runs real
long-lived servers, which the incident pipeline and the ~3-minute repo analysis
need.

Everything is described in [`render.yaml`](../render.yaml) (a Render Blueprint).

## Services it creates

| Service | Type | Notes |
|---------|------|-------|
| `volt-tackle-db` | Postgres | managed, **free** (expires after 90 days) |
| `volt-tackle-qdrant` | Private service (Qdrant) | ⚠️ **paid** instance — or use free Qdrant Cloud (below) |
| `volt-tackle-api` | Web (Fastify) | runs migrate → seed → qdrant-bootstrap → server |
| `volt-tackle-web` | Web (Next.js) | the dashboard + landing page |

## 1. Deploy the Blueprint

1. Push this repo to GitHub (already done).
2. Render dashboard → **New → Blueprint** → select this repo → **Apply**.
3. Render provisions all four services from `render.yaml`.

## 2. Two manual URL steps (required)

Render can't know the cross-service URLs until the services exist, so wire them
once after the first deploy:

1. **API → allow the frontend (CORS).** On `volt-tackle-api` → Environment, set
   `CORS_ORIGIN` = your web URL, e.g. `https://volt-tackle-web.onrender.com`.
2. **Web → point at the API.** On `volt-tackle-web` → Environment, set
   `NEXT_PUBLIC_API_BASE_URL` = your API URL, e.g.
   `https://volt-tackle-api.onrender.com`. This is **baked into the build**, so
   after setting it, **Manual Deploy → Deploy latest commit** on the web service.

After that, the dashboard talks to the API and the whole app works.

## 3. Qdrant: two options

- **Paid, self-contained (default in `render.yaml`):** the `volt-tackle-qdrant`
  private service. Private services require a paid instance (~$7/mo). Data is
  ephemeral (no disk) and is re-indexed from Postgres on each API deploy.
- **Free, managed (recommended):** create a free 1 GB cluster at
  [Qdrant Cloud](https://cloud.qdrant.io). Then **delete the `volt-tackle-qdrant`
  block** from `render.yaml` and, on the API service, set:
  - `QDRANT_URL` = your cluster URL (e.g. `https://xyz.qdrant.io:6333`)
  - `QDRANT_API_KEY` = your cluster key

## 4. Enable the real AI providers (optional)

The app **runs fully offline with mocks by default** — no keys needed. To turn
on the real integrations, set the secret(s) on `volt-tackle-api` → Environment,
then flip the matching provider var, and redeploy the API:

| To enable | Set secret | Set provider |
|-----------|-----------|--------------|
| Featherless (reasoning) | `FEATHERLESS_API_KEY` | `LLM_PROVIDER=featherless` |
| Enkrypt (safety) | `ENKRYPT_API_KEY` | `SAFETY_PROVIDER=enkrypt` |
| GitAgent + Lyzr (repo analysis) | `GITHUB_TOKEN`, `LYZR_API_KEY`, `GITAGENT_LYZR_AGENT_ID` | `REPO_CONTEXT_PROVIDER=gitagent` |

`JWT_SECRET` is auto-generated by Render. Never commit secrets — set them in the
dashboard.

## Honest caveats (free tier)

- **Free web services sleep** after ~15 min idle and cold-start in ~50s. The
  first request after idle is slow.
- **Repo analysis (~3 min)** can exceed a free instance's request/CPU limits —
  use a paid instance for `volt-tackle-api` if you demo that feature live.
- **First deploy ordering:** the API's `qdrant:bootstrap` may run before Qdrant
  is ready; it's best-effort (`|| true`) and self-heals on the next deploy, or
  hit **Manual Deploy** on the API once Qdrant is up.
- **Free Postgres expires after 90 days.**
